# Database Migrations — Alembic

Gervis uses **Alembic** for database schema migrations with SQLAlchemy async engine.
Migration files live in `migrations/versions/`.

---

## Setup (already done)

The following files were created during initial setup — no action needed:

```
gervis-backend/
├── alembic.ini              ← Alembic config (points to migrations/)
└── migrations/
    ├── env.py               ← Async migration runner, loads settings from .env
    ├── script.py.mako       ← Template for new migration files
    └── versions/            ← Generated migration files go here
```

---

## Common Commands

```bash
# Apply all pending migrations (create/update tables)
uv run alembic upgrade head

# Generate a new migration from model changes
uv run alembic revision --autogenerate -m "description of change"

# Check current migration version in the database
uv run alembic current

# See migration history
uv run alembic history

# Roll back one migration
uv run alembic downgrade -1

# Roll back all migrations
uv run alembic downgrade base
```

---

## Workflow: Adding a New Model or Column

1. Edit `db/models.py` — add or modify a SQLAlchemy model.
2. Generate the migration:
   ```bash
   uv run alembic revision --autogenerate -m "add jobs table"
   ```
3. Review the generated file in `migrations/versions/` — always check autogenerated SQL.
4. Apply it:
   ```bash
   uv run alembic upgrade head
   ```

---

## How It Works

`migrations/env.py` loads your `.env` via `config.settings` and uses the `DATABASE_URL`
to connect. It runs async migrations using `asyncpg` directly — no sync driver needed.

The `alembic.ini` has a placeholder URL (`sqlalchemy.url = placeholder`) which is always
overridden at runtime by `env.py`.

---

## Known Gotchas

- **asyncpg URL format**: use `postgresql+asyncpg://...?ssl=require`, not `sslmode=require`.
- **ALLOWED_ORIGINS in .env**: must be a JSON array — `["http://localhost:3000"]`, not a plain string.
- **FIREBASE_SERVICE_ACCOUNT_JSON**: if not yet configured, set to `{}` to avoid parse errors.
- Alembic autogenerate does **not** detect every change (e.g. check constraints, some indexes).
  Always review generated files before applying.
